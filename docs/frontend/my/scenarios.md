# 계정 설정 시나리오

## 관련 페이지 / API

| 경로 | 역할 |
|------|------|
| `/my` | 계정 정보 확인 · 로그아웃 · 회원 탈퇴 |
| `GET /members/{id}` | 회원 정보 조회 |
| `DELETE /members/{id}` | 회원 탈퇴 (cascade 삭제) |

---

## 시나리오 1 — 페이지 진입

1. `localStorage["kkachi_member_id"]` 없으면 → `/join` replace redirect
2. `GET /members/{id}` 호출
   - 성공: 이름·이메일 표시
   - 404 (탈퇴·만료): localStorage 제거 → `/join` replace redirect

---

## 시나리오 2 — 로그아웃

1. "계정 정보" 헤더 우측 "⏻ 로그아웃" 버튼 클릭
2. `localStorage["kkachi_member_id"]` 제거
3. `/` push redirect

---

## 시나리오 3 — 회원 탈퇴

1. "탈퇴하기" 버튼 클릭 → 이메일 확인 폼 표시
2. 가입 이메일 입력 → 일치해야 "탈퇴 확인" 활성화
3. `DELETE /members/{id}` 호출
4. 성공: localStorage 제거 → `/` push redirect
5. 실패: "탈퇴 처리 중 오류가 발생했습니다." 표시, 폼 유지

### Cascade 삭제 범위
`members` → `profiles` → `analyses` · `daily_fortunes` · `compatibilities`
탈퇴 후 동일 이메일로 재가입 시 완전히 새 계정으로 시작.
