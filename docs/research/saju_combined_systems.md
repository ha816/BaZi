## 1. 시장 현황 — 결합 서비스 벤치마킹

### 1-1. 국내 서비스

#### MBTIID (mbtiid.com)
- **컨셉**: 사주 + MBTI + 퍼스널컬러 + 연애유형을 단일 플랫폼에서 제공
- **결합 방식**: 병렬 독립 운영. 생년월일·출생시·출생지로 사주를 분석하고, 별도로 16문항 MBTI 테스트를 진행한 뒤 두 결과를 함께 궁합 점수화
- **핵심 UI**: 퍼센트 바로 각 MBTI 축 강도를 시각화. 50%=초록, 100%=빨강
- **평가**: 진정한 교차 해석은 없음. 두 체계를 나란히 놓고 "정밀 궁합"이라는 라벨로 포장한 수준

#### 포스텔러 소름돋는사주 sodotsa (sodotsa.forceteller.com/interactive/sodotsa6)
- **컨셉**: 생년월일시만 입력하면 사주에서 MBTI 유형을 도출해주는 인터랙티브 이벤트 콘텐츠
- **결합 방식**: 단방향 파생형 — 사주 데이터 → MBTI 라벨 출력. 내부 매핑 로직은 비공개
- **바이럴 특성**: SNS 공유용 카드 형태로 설계. 접근성이 높고 결과 공유 유도
- **평가**: 학술 기반이 아닌 콘텐츠 마케팅 용도. 신뢰도보다 화제성에 최적화

#### 헬로우봇 (hellobot.co)
- **컨셉**: 사주 + 타로 + 별자리 + 궁합 통합 플랫폼 (누적 600만 다운로드)
- **결합 방식**: 카테고리 병렬 나열. 각 체계를 독립 탭으로 분리하여 사용자가 선택
- **MBTI 연동**: 직접 결합은 없음. 개별 콘텐츠로 분리 운영
- **수익 모델**: 유료 전환 과도로 인해 사용자 이탈 증가 (Threads 유저 리뷰)

#### 미리소개팅 (데이팅앱)
- **컨셉**: 사주 궁합 기반 매칭에 MBTI 궁합 기능 추가
- **결합 방식**: MBTI 입력 → MBTI 궁합 좋은 상대 추천. 사주와 MBTI는 각각 독립 필터
- **특이점**: 매일 1시간 무료 MBTI 궁합 추천. 사주 기반 "오늘의 인연"과 병렬 운영
- **시사점**: 두 체계를 교차해석이 아닌 독립 필터로 구현하는 것이 가장 현실적인 초기 접근

#### 사주닷cc (saju.cc)
- **컨셉**: AI 사주풀이 + MBTI 만세력 결합
- **결합 방식**: 만세력 결과에 사용자 MBTI를 병기하여 공유 카드 생성

### 1-2. 해외 서비스

#### Mirror (mirror-hq.com)
- **컨셉**: 10개 이상의 성격 시스템을 단일 프로필로 통합 — MBTI, 에니어그램, Big Five, DISC, 휴먼 디자인, 서양/베다/중국/마야 점성술, 수비학
- **결합 방식**: 생년월일·시·장소 1회 입력 → 모든 시스템 동시 계산. AI가 교차 해석 제공
- **UI**: 시스템별 탭 + AI 채팅 ("Ask") 으로 통합 인사이트 질문 가능
- **수익 모델**: 무료 기본 차트 / 유료 상세 보고서 + AI 채팅 무제한
- **사용자 반응**: "개별 시스템에 회의적이었는데 합치니 더 맞는 것 같다" (Product Hunt 리뷰)
- **시사점**: 통합 자체가 신뢰도를 보완. 어느 한 체계를 믿지 않는 사용자도 여러 체계의 교집합을 신뢰

#### Sajume (sajume.com)
- **컨셉**: 사주(SJPJ)를 "오행 균형 기반 다차원 MBTI"로 포지셔닝
- **결합 방식**: 사주를 MBTI의 대안으로 제시 — "서양 점성술은 태양 별자리에, 한국 시스템은 일주(日主)에 집중"
- **특징**: MBTI 4축과 직접 매핑하지 않고 사주를 독립 성격 진단 프레임으로 브랜딩
- **시사점**: 결합이 아닌 "사주가 MBTI보다 더 정밀한 성격 분석"이라는 포지셔닝도 유효

#### Co-Star (costarastrology.com)
- **컨셉**: 서양 점성술 특화. NASA 데이터 + AI로 초개인화 일일 호로스코프
- **MBTI 결합**: 없음. 점성술 단독 심층화 전략
- **시사점**: 단일 체계를 극도로 정교하게 만드는 전략도 차별화가 됨

## 2. MBTI x 사주 결합 — 학술적 근거

### 2-1. 핵심 연구 결과

한국 학술지에 발표된 3편의 논문이 사주와 MBTI의 상관관계를 실증 연구했다.

**논문 1: "MBTI와 사주명리의 연관성 고찰 — 외향성과 내향성을 중심으로"**
(이남호·김만태, 인문사회 21, 2018)
- 핵심 발견: E/I 축과 오행 분포의 상관관계 확인
  - **木·火 오행 우세 → 외향성(E)** 경향
  - **金·水 오행 우세 → 내향성(I)** 경향
  - 목일간(甲·乙)·봄생(寅·卯·辰월) 사람에서 외향형 비율이 높게 나타남
- 한계: 소표본(각 10명). 나머지 3축(S/N, T/F, J/P) 연구는 후속 과제로 제안

**논문 2: "사주명리학의 8가지 성격유형과 MBTI 기능별 8가지 성격유형의 상관연구"**
(scienceon.kisti.re.kr)
- 핵심 발견: 십신(十神)과 MBTI 인지기능의 대응관계

| 십신(十神) | MBTI 인지기능 | 유의미성 |
|----------|-------------|---------|
| 상관(傷官) | 외향감정 (Fe) | 높음 |
| 정관(正官) | 외향사고 (Te) | 높음 |
| 식신(食神) | 내향감정 (Fi) | 유의미 |
| 정재(正財) | 내향감각 (Si) | 유의미 |
| 편재(偏財) | 외향감각 (Se) | 유의미 |
| 편관(偏官) | 사고유형 전반 | 부분적 |
| 정인(正印) | - | 유의미하지 않음 |
| 편인(偏印) | - | 유의미하지 않음 |

- 한계: 정인·편인(직관 관련)의 대응성이 불명확. 직관(N) 축은 사주 도구로 포착이 어려움

**논문 3: "명리학과 MBTI의 성격심리에 관한 연관성 연구"**
(인문사회 21, 2023)
- 오행 음양 비율 세력으로 E/I 구분에 통계적 유의미성 확인
- 오행 분포 5개 항목이 복합적으로 E/I에 영향

**단행본 참고**
- 허은경, 「사주×MBTI 성격 사전」: 일간(日干) + 월지(月支) 조합으로 성격 기질 도출. 사주가 MBTI를 추론하는 방향
- TVING 오리지널 "MBTI vs 사주" (2023): 실험 예능. 문화적 대중 관심 확인

### 2-2. 학술 연구 요약 및 함의

**확인된 매핑 (근거 있음)**
- 木·火 우세 → E 성향 / 金·水 우세 → I 성향
- 상관(傷官) → 외향감정(Fe) / 정관(正官) → 외향사고(Te)
- 식신(食神) → 내향감정(Fi) / 정재(正財) → 내향감각(Si)

**불확실한 매핑 (연구 부족)**
- S/N 축: 화일간·금일간이 감각형(S) 경향이 있다는 결과 있으나 표본 불충분
- J/P 축: 아직 체계적 연구 없음
- 정인·편인(正印·偏印) → 직관(N)은 상관성 불명확

**결론**: 4축 중 E/I는 오행 분포로 어느 정도 근거 있는 추론이 가능하다. 나머지 3축은 추론이지 검증된 매핑은 아니다. 서비스에서는 이 한계를 투명하게 노출하는 것이 신뢰도에 유리하다.

## 3. 결합의 함정과 주의점

### 3-1. 신뢰도 저하 위험

- MBTI 자체가 재검사 시 유형이 바뀌는 문제가 있음. "사주로 도출한 MBTI"는 이중 불확실성
- 사용자가 자신이 알고 있는 MBTI(예: INFJ)와 사주에서 도출된 MBTI(예: ENFP)가 다를 때 혼란 발생
- 해결책: 사주에서 MBTI를 "도출"하는 것이 아니라 사주 성격 특성을 MBTI 언어로 "번역"하는 프레임 사용 ("당신의 사주에서 보이는 특성은 MBTI의 이런 면과 닮았습니다")

### 3-2. 두 입력 방식 비교

| 방식 | 장점 | 단점 |
|------|------|------|
| 사용자가 MBTI 직접 입력 | 정확한 MBTI 활용 가능. 입력 마찰 낮음 | 사주와 교차 해석하려면 서비스가 매핑 로직 내장 필요 |
| 사주에서 MBTI 추론 | 추가 입력 없음. 바이럴 콘텐츠로 유용 | 학술적 근거 불완전. 사용자가 자신의 MBTI와 다를 때 이탈 |

**권장**: 두 방식을 목적에 따라 분리 운영
- 바이럴 진입 콘텐츠 → "사주로 보는 나의 성격 유형" (MBTI 언어 차용, 추론)
- 심층 분석 → 사용자가 MBTI 직접 입력 → 사주와 교차 인사이트 제공

### 3-3. 프레이밍 전략

"사주는 운명, MBTI는 성격"이라는 이분법 대신:
- **사주 = 타고난 기질의 지도**. 변하지 않는 자원과 경향
- **MBTI = 현재 행동 패턴**. 환경에 따라 변할 수 있는 표현 방식
- 결합 메시지 예: "당신의 木 기질(갑목)은 MBTI의 E 특성과 닮아 있어요. 두 체계 모두 당신이 에너지를 밖으로 발산하는 사람이라고 말하고 있습니다."

## 4. 사주까치 적용 방향 — 기능 컨셉 3가지

### 컨셉 A: "사주 렌즈로 보는 나" 성격 카드 (바이럴 진입용)

**핵심 컨셉**
사주에서 계산된 오행 분포·일간·월지·십신 분포를 MBTI 언어로 번역한 짧은 성격 카드. "나는 사주로 보면 ENFP 계열"이 아니라 "나의 사주에서 읽히는 성격 DNA"라는 프레임.

**활용 데이터**
- `natal.element_stats` (오행 분포) → E/I 추론
- `natal.day_stem` (일간) → 갑/을/병/정/무/기/경/신/임/계 성격 특성
- `natal.sipsin` (십신 분포) → 상관 多=Fe 성향 / 정관 多=Te 성향 등

**규칙 기반 매핑 (백엔드 구현)**
```
木·火 합산 > 金·水 합산 → E 경향
金·水 합산 > 木·火 합산 → I 경향
土 우세(3개 이상) → 중간형

상관(傷官) 최다 → 감정표현 중심 (Fe-like)
정관(正官) 최다 → 원칙·논리 중심 (Te-like)
식신(食神) 최다 → 내면 충실 (Fi-like)
정재(正財) 최다 → 현실·경험 중심 (Si-like)
편재(偏財) 최다 → 외부 자극 추구 (Se-like)
```

**UI 노출 위치**
- `PersonalityTab.tsx` 안 새 카드 섹션 — "사주가 말하는 나의 성격 DNA"
- 공유 버튼 포함. 카드 이미지 생성 → SNS 바이럴 유도

**구현 위치**
- `src/kkachi/application/interpreter/personality.py` — `PersonalityInterpreter.__call__()` 확장
- `interpretation.py` — `NatalResult.narratives` dict에 `"mbti_like"` 키 추가
- 프론트: `frontend/src/components/tabs/PersonalityTab.tsx` 신규 카드 추가

**구현 난이도**: 낮음 (Rule 기반, 추가 API 없음)
**리스크**: 사용자가 자신의 실제 MBTI와 다른 결과를 받을 수 있음 → 카피를 "추론" 이 아닌 "사주에서 읽히는 성향"으로 명확히 표현할 것

### 컨셉 B: 사용자 MBTI 입력 → 교차 인사이트 (심층 분석 강화)

**핵심 컨셉**
심층 분석 입력 단계에서 MBTI를 선택입력 받고, 사주 해석 결과와 교차하여 "두 체계가 동시에 가리키는 것"을 강조. Mirror 앱의 "교집합 신뢰도" 전략을 차용.

**활용 데이터**
- 사용자 입력 MBTI (예: INTJ)
- `natal.element_stats`, `natal.sipsin`, `natal.sinsal`, `natal.yongshin_info`

**교차 해석 예시**
- 사용자 MBTI가 INTJ, 사주에 金·水 우세 + 정관(正官) 많음:
  → "사주와 MBTI 모두 당신이 체계적이고 목표 지향적인 사람임을 가리킵니다. 정관의 원칙성과 INTJ의 전략적 사고가 일치하네요."
- 사용자 MBTI가 ENFP, 사주에 金·水 우세:
  → "MBTI는 외향적 감각을 보여주지만, 사주의 金·水 기운은 내면에 깊은 사유와 신중함이 있음을 말합니다. 겉으로는 활발하지만 혼자 있는 시간이 필요한 이유가 여기에 있을 수 있어요."

**UI 노출 위치**
- `AnalysisForm.tsx` — "정밀 설정" collapsible 안에 MBTI 선택 드롭다운 추가
- `PersonalityTab.tsx` — "MBTI × 사주 교차 분석" 섹션 (입력 없으면 숨김)

**구현 위치**
- `frontend/src/components/AnalysisForm.tsx` — MBTI 선택 UI (선택입력)
- `frontend/src/lib/api.ts` — 요청 body에 `mbti?: string` 추가
- `src/kkachi/adapter/inner/saju_controller.py` — 요청 파라미터 수신
- `src/kkachi/application/interpreter/personality.py` — MBTI 교차 해석 로직 추가

**구현 난이도**: 중간 (입력 흐름 변경 + 새 해석 로직)
**리스크**: 선택입력이므로 입력하지 않는 사용자가 많을 수 있음. 인센티브 제공 필요 (입력 시 추가 인사이트 카드 노출 등)

### 컨셉 C: "사주 × MBTI 궁합 시너지" 궁합 탭 확장

**핵심 컨셉**
기존 오행 기반 궁합에 MBTI 궁합을 레이어로 추가. "두 사람의 사주도 잘 맞고 MBTI도 잘 맞는다" 또는 "사주는 잘 맞는데 MBTI는 충돌이 있어서 서로 맞춰가는 노력이 필요하다"는 식의 복합 궁합 내러티브 생성.

**활용 데이터**
- 기존 `CompatibilityResult` 오행 점수
- 두 사람의 MBTI (선택 입력)
